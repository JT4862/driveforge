Grading thresholds

{% if saved_panel == "grading" %}saved{% endif %}
{# v0.8.0+ Buyer-transparency grading rules. Collapsible sections so the main grading panel doesn't become a wall of inputs. Each section has its own enable/disable toggle so operators can switch off an entire category while tuning others. #}
Age-based ceilings (power-on hours)

A drive can accumulate clean SMART counters while still being at the end of its design life. These ceilings cap grade by age: demoted but not auto-failed unless the hard threshold below is set.

Workload ceilings (lifetime writes vs rated TBW)

Compares lifetime host writes against the drive's class-rated workload. Classes are auto-detected from model + transport + rotation rate; ambiguous drives default to the consumer tier for their media type (tighter ceiling — safer for refurb grading).

Rated lifetime writes per drive class (TB): Sane defaults based on typical market 5-year design life (550 TB/yr enterprise HDD, 55 TB/yr consumer HDD, etc). Adjust for your specific SKUs if you know better.

SSD wear ceilings

SSDs expose a 0–100% "wear indicator" (NVMe percentage_used or SATA attrs 233/177/231/169). HDDs have no equivalent — these rules no-op on spinners.

Error-based auto-fail / ceiling rules

Individual drive-reported error signals. All disableable individually so operators can soften specific flags without turning off the category.

Brother QL-820NWB and other network-capable QL models listen on TCP 9100 by default. Find the printer's IP from its LCD menu (WLAN → TCP/IP → IP Address) or your DHCP lease list. Test Print below verifies reachability + data dispatch in one click.

When enabled, DriveForge auto-prints the cert sticker as soon as a drive's full pipeline finishes grading. Applies to pass AND fail tiers (both produce a sticker with the grade). {# v0.6.2+ template-mode hint. Brother QL printers ship (or get left) in "template mode" — a Brother-proprietary ESC/P command mode for their P-touch Editor software. Not printing? Check raster mode — Brother QLs default to template mode

Brother QL printers have two command modes: raster (what DriveForge sends) and template (Brother's P-touch Editor pre-stored templates). A printer in template mode silently discards raster data — you see "Printing was successful" in the daemon log but nothing comes out.

Switch to raster mode via the printer's physical menu: Setup → Command Mode → Raster on the QL-820NWB (path varies by model — consult the Brother user guide). Power-cycle the printer once changed.

Purge legacy fail rows (v0.5.1 migration)

Before v0.5.1, grade="fail" conflated two distinct outcomes: real drive-fail (grading rules failed the drive) and pipeline-error (daemon broke mid-run). v0.5.1 splits these into F and error respectively. Legacy rows can't be retroactively classified; purging them lets affected drives auto-re-test cleanly with the new vocabulary.

Only affects rows with grade="fail" — does not touch A/B/C, F, error, or aborted runs. Cascades to SMART snapshots + telemetry samples for those runs.
