{# v0.10.0+ Role selection. Most installs are standalone; operator mode unlocks fleet aggregation when the user has more than one DriveForge box. Agent mode is configured via the `driveforge fleet join` CLI on the agent itself, not here — so it's intentionally absent from this form. #}
How will this DriveForge install be used?

Setting up an agent? Install DriveForge on the agent box the normal way (leave this wizard in standalone mode), then on the operator node go to Settings → Agents, generate an enrollment token, and paste the driveforge fleet join … one-liner on the agent's console. Agents don't need the web UI.

{# v0.10.0+ Hostname confirmation. install.sh uniquifies the hostname on first install (driveforge-), but operators usually want something friendlier like "driveforge-r720". Showing the auto-generated name here + letting them rename in one step fixes the historical "everything is driveforge.local on my LAN" papercut. #}
Hostname

Published as {{ current_hostname }}.local via mDNS. The installer generated a unique name from this box's MAC address so it doesn't collide with other DriveForge boxes on your LAN. Rename it now to something friendlier (e.g. driveforge-r720) if you'd like; you can also change it later from Settings. Letters, digits, and hyphens only.

Every optional integration can be skipped now and configured later in Settings. DriveForge works end to end with local-only features — no external services are required.

  • Auto-detect attached drives and the server's network info

  • Review grading thresholds (defaults are fine for most homelab use)
  • Connect optional integrations (outbound webhook, Cloudflare Tunnel) or skip
